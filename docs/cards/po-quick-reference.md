# Project Officer · Quick Reference (v4.1)

## What you do here every day
1. **My Day** — see returned items, due liquidations, reminders.
2. **Advance Requests** — raise, edit, resubmit. Liquidate when spent.
3. **Action Points** — clear yours; they age into RED at +14 days.

## Raising an AR — checklist before you click Submit
- [ ] DIP code matches the activity
- [ ] Amount is what your concept note says
- [ ] Concept note is attached
- [ ] Due liquidation date is realistic (Manual: ≤ 14 days post-activity)
- [ ] No other unliquidated advance on this activity (Rule 50)

## Liquidation — three buttons
| Button | When |
|---|---|
| **Liquidate full** | Spent = advance, no variance |
| **Liquidate with variance** | Spent ≠ advance — note required |
| **Return funds** | Activity cancelled; refund EFT ref needed |

## When something is RETURNED
The reason is at the top of the AR/PV. Edit, fix, resubmit. Don't argue in the system — argue in person, then resubmit.

## Common error messages
| Message | Means |
|---|---|
| `Outstanding advance ARxxxx…` | Liquidate the old one first |
| `Three-way match required` | Attach LPO + GRN ids on the PV |
| `WTEC expired` | Vendor's tax cert lapsed — Procurement renews |
